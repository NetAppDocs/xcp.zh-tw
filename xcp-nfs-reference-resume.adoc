:linkattrs:

[role="lead"]
XCP NFS `resume` 命令會指定目錄索引名稱或編號，以重新啟動中斷的複本作業。目錄索引名稱或上一個複本作業的編號位於中 `<catalog path>:/catalog/indexes` 目錄。

.語法
[source, cli]
----
xcp resume -id <catalog_name>
----

IMPORTANT: 。 `-id <catalog_name>` 參數是必要的 `resume` 命令。

下表列出 `resume` 參數及其說明。

[cols="2*"]
|===
| 參數 | 說明 


| <<resume_id,繼續 -id  ； catalog_name>  | 指定先前複本索引的目錄名稱。這是 RESUME 命令所需的參數。 


| <<resume_bs,繼續 - bs ； n[k]>  | 指定讀寫區塊大小（預設值： 64K ）。 


| <<resume_dircount,繼續 -dircount  ； n[k]>  | 指定讀取目錄時的要求大小。 


| <<resume_parallel,繼續 -parallel  ； n>  | 指定並行批次處理程序的最大數目（預設值： 7 ）。 


| <<resume_preserve_atime,恢復 - 保留 - 保留>> | 將來源上的所有檔案還原至上次存取的日期。 


| <<resume_s3_insecure,繼續 -s3.insecure>> | 提供選項、可在 S3 Bucket 通訊中使用 HTTP 而非 HTTPS 。 


| <<resume_s3_endpoint,繼續 -S3.Endpoint  ； S3_Endpoint URL>  | 以 S3 Bucket 通訊的指定 URL 覆寫預設的 Amazon Web Services （ AWS ）端點 URL 。 


| <<resume_s3_profile,繼續 -s3.profile   | 從 AWS 認證檔案指定用於 S3 儲存區通訊的設定檔。 


| <<resume_s3_noverify,繼續 -s3.noverify >> | 覆寫 S3 Bucket 通訊的 SSL 認證預設驗證。 
|===

== 恢復 - 保留 - 保留

使用 `-preserve-atime` 參數 `resume` 命令將所有檔案還原至來源上上次存取的日期。

。 `-preserve-atime` 參數會將存取時間重設為 XCP 讀取作業之前設定的原始值。

.語法
[source, cli]
----
xcp resume -id <catalog_name> -preserve-atime
----
